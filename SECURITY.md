# Security Policy

## Supported version

Security fixes are applied to the latest published release.

## Reporting

Do not publish secrets, private media, local paths, or exploit details in a
public issue. Use GitHub's private vulnerability reporting feature for the
repository. If that feature is unavailable, contact the repository owner
privately through the contact method shown on their GitHub profile.

Include:

- the affected version and file;
- a minimal reproduction without private data;
- the potential impact;
- any suggested mitigation.

Do not include API keys, tokens, passwords, personal conversations, customer
files, or unlicensed media in a report.
